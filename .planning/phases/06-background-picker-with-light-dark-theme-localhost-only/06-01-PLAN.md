# Plan 06-01: Background picker with light/dark theme (single-wave implementation)

## Objective

Ship a runtime, user-controllable background picker — floating widget on every page, three options (WebGL Shader / Dark / Light), persisted via localStorage and synced via Supabase. Includes a real `next-themes`-driven light/dark theme refactor for major surfaces. All work is localhost-only — source-code changes stay uncommitted per the project's existing convention.

<action>
- Create `src/components/BackgroundProvider.tsx` (client component):
  - Imports `setItemAndNotify` from `@/lib/sync-event` and `useTheme` from `next-themes`.
  - State: `background: 'shader' | 'dark' | 'light'` (default `'shader'`).
  - On mount, read `localStorage.getItem('admitedge-bg-preference')` and apply.
  - Subscribe to the `profile-source-updated` window event; when `event.detail.key === 'admitedge-bg-preference'`, re-read and update state (and call `setTheme`).
  - Expose `{ background, setBackground }` via context.
  - `setBackground(value)` calls `setItemAndNotify('admitedge-bg-preference', value)` AND `setTheme(value === 'light' ? 'light' : 'dark')`.
  - Export `useBackground()` hook.
- Rewrite `src/components/AuroraBackground.tsx`:
  - Remove the `USE_DOTTED_SURFACE` env constant.
  - Mark as client component if not already.
  - Read `useBackground()`. For `'shader'`, render `<WebGLShader />`. For `'dark'` and `'light'`, render no background layer (the themed body surface is the background).
  - Keep the `bg-bgBase text-textPrimary` (or token-driven) wrapper styling.
- Mount `<BackgroundProvider>` inside `src/app/layout.tsx` so it wraps `AppShell` (above or below ThemeProvider — BackgroundProvider must be a child of ThemeProvider to use `useTheme`).
- Rewrite `src/app/page.tsx`'s background block to consume the same `useBackground()` provider, dropping the env-gated branch entirely (already partially done in Task 1).
</action>
