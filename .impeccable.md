# AdmitEdge — Design Context

This file is consumed by the `/impeccable` skill on every design pass. Edit
it whenever the brand or audience shifts.

## Design Context

### Users

Elite high-school applicants — the kind with UW 4.0, 1540+ SAT, 6+ APs, and
distinguished EC accomplishments (national competition placements, first-author
publications, founder credits). Often using the app late at night during
focused planning sessions on a laptop or tablet, sometimes alongside a parent
or counselor.

Job-to-be-done: build a confident college list, decide where to spend ED, and
stop second-guessing the math. They want a sober second opinion, not a
gamified matcher.

### Brand Personality

Three words: **precise, sober, considered.**

Voice: an admissions consultant with strong opinions and primary-source
citations. Never breathless. Never "your dream school is waiting!" Treat the
user like the intellectual peer they are.

Should evoke confidence, calm, intellectual seriousness.
Should NOT evoke gamification, pastel hype, mascots, or casual SaaS energy.

### Aesthetic Direction

Editorial luxury / restrained monochrome with sharp typographic hierarchy.

**References (positive):**
- Bloomberg Terminal — for tabular numerics and dense without noise
- Apple Newsroom mastheads — for serif display + sans body discipline
- Linear app — for calm density
- FT.com — for editorial chrome and small-caps eyebrow labels
- Klim Type Foundry's site — for the "every detail considered" feel

**Anti-references:**
- Generic AI college-matchmaker SaaS (pastel gradients, cartoon mascots)
- "Find your dream school!" hype copy
- Big colorful icons floated above every heading
- Gradient-clipped text used for emphasis
- Cyan-on-dark with neon glows

**Theme**: monochrome default (already configured in globals.css), with dark
and light alternates. All three should feel premium and intentional.

### Design Principles

1. **Numbers are the hero.** The letter grade, chance %, AI score, tier counts
   — render them as masthead figures with the display serif at scale, not as
   data labels. Tabular figures, generous size contrast.

2. **Dense without noise.** Tight rhythm, deliberate whitespace, no repeated
   card grids. Break the grid intentionally for emphasis. Section eyebrows in
   small all-caps tracking, body in sans, hero moments in display serif.

3. **Editorial copy hierarchy.** Eyebrow → headline → standfirst → body. Match
   how a print magazine introduces a feature, not how a SaaS dashboard
   introduces a section.

4. **Restraint over decoration.** Glass cards stay subtle. No gradient text,
   no neon, no left-stripe accent borders. Color is used semantically (tier
   dots, amber disclaimer) and never for decoration.

5. **Honest tone.** Methodology link is always one click away. "Estimates,
   not predictions" copy stays explicit.

### Typography

- **Display**: Young Serif (already loaded via `--font-display`). Use for hero
  numbers and section mastheads.
- **Body**: Geist Sans (`--font-geist-sans`). Use for everything else.
- **Numerics**: Geist Mono (`--font-geist-mono`) for tabular figures only —
  not as a "developer aesthetic" choice.

Banned (per impeccable rules): Inter, Fraunces, Newsreader, Lora, Crimson,
Playfair, Cormorant, IBM Plex, Space Mono, Space Grotesk, DM Sans, Outfit,
Plus Jakarta, Instrument, Syne. The current Geist + Young Serif pairing
predates this rule and is acceptable; future additions must look elsewhere.

### Color

- Already on OKLCH-friendly tokens via `--text-primary`, `--accent`, etc.
- Tier dots (emerald/blue/amber/orange/red/zinc) are semantic and stay.
- No gradient text. The existing `.text-gradient` class is just `color: white`,
  not a gradient — keep using it where the contract is "primary display color"
  but rename intentions in code so it doesn't read as a gradient marker.
- Tinted neutrals already in place.

### Spacing & layout

- Tailwind 4 with the 4pt scale.
- Embrace asymmetric layouts. Don't center everything.
- Use the existing `--ease-out` (Emil Kowalski) and `--duration-*` tokens for
  motion.
